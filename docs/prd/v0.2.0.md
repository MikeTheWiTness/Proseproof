# Proseproof v0.2.0 PRD

## 概述

v0.2.0 在 v0.1.0 的「可组合 AI 校对流水线」基础上，引入 **Strategy + Middleware
双层架构**、**大纲驱动的智能分割**、**文档级内容审查**和**断点续传**四大能力，
将 Proseproof 从一个「能跑通的校对工具」升级为一个「可深度定制的校对引擎」。

核心设计原则：**单流水线，极致客制化**。

## 架构变更

```
v0.1.0                          v0.2.0
───────                         ───────
Pipeline                        Pipeline
├── Convert                     ├── Convert ── Strategy (不变)
├── Split                       ├── Split ── Strategy: heading|smart|deep|manual|none
├── Proofread ── 单一方法        │                └── 产出 _outline.json
├── Typeset                      ├── 结构审查 ── Python 规则（如果有大纲）
└── Compile                      ├── Proofread ── Strategy: default
                                 │     └── Middleware Chain
                                 │           ├── PreCheck (pre)
                                 │           ├── [LLM 校对]
                                 │           │     └── 顺手产出 summary
                                 │           └── Similarity (post)
                                 ├── 内容审查 ── Light | Full | Off
                                 ├── Typeset ── Strategy (不变)
                                 └── Compile ── Strategy (不变)
```

## CLI 接口

### `proseproof split`

```
proseproof split <input>
    --mode smart              # heading|smart|deep|manual|none (默认 smart)
    --outline-depth 4         # 大纲提取深度
    --split-by-pattern <re>   # 逃生舱：自定义正则
    --numbered-hints <re>     # smart 模式补充编号提示
```

### `proseproof proofread`

```
proseproof proofread <frag_dir>
    --react                   # ReAct 模式
    --middleware pre_check,similarity  # 中间件链（默认 pre_check,similarity）
```

### `proseproof run`

```
proseproof run <input>
    --split-mode smart        # 分割模式
    --middleware pre_check,similarity
    --review light            # light|full|off（默认 light）
    --yes                     # 不暂停，全自动
    --resume                  # 断点续传
    --output-dir <path>
```

### 行为

- `run` 严格停止：任何片段失败立即终止流水线，清单记录进度。
- `run --resume`：跳过 `completed` 且 MD5 一致的片段，从断点继续。
- `--review light`（默认）：大纲 + 摘要审查，成本低。
- `--review full`：全文审查，用户主动选择。
- `--review off`：跳过内容审查。

## Profile 配置

`config.json` 新增字段：

```json
{
  "split": {
    "mode": "smart",
    "outline": {
      "max_depth": 4,
      "numbered_patterns": ["^\\d+[.、．]", "^[(（]\\d+[)）]"],
      "numbered_hints": []
    }
  },
  "proofread": {
    "strategy": "default",
    "middleware_chain": [
      {"name": "pre_check", "enabled": true},
      {"name": "similarity", "enabled": true, "threshold": 0.85}
    ]
  },
  "review": {
    "structural": {"enabled": true},
    "content": {"mode": "light"}
  }
}
```

| 产物 | 路径 | 产出阶段 |

| 校对数据 | `output/{doc}/frag_NNN/_校对数据.json` | Proofread |

| 状态清单 | `output/{doc}/.proofread_manifest.json` | Proofread |
| 内容审查报告 | `output/{doc}/_review_report.json` | Review |

## API 接口（Python）

第三方扩展可实现的接口：

```python
class SplitStrategy(Protocol):
    def split(self, content: str, config: dict) -> list[Fragment]: ...

class ProofreadMiddleware(Protocol):
    name: str
    phase: Literal["pre", "post"]
    def process(self, ctx: ProofreadContext) -> MiddlewareResult: ...

class BaseProfile:
    def register_middleware(self) -> dict[str, ProofreadMiddleware]: ...
```
