# Proseproof v0.3.0 PRD

## 概述

v0.3.0 是 **质量加固 + 通用化** 版本。在 v0.2.0 完成架构闭环之后，发现以下问题：

- **死代码**：约 3,200 行代码从未被 import 或调用，部分有接入价值
- **重复代码**：4 组函数/常量在两处重复定义
- **错误处理缺口**：6 处静默数据丢失、profile 加载未保护、CLI 选项无效
- **垂直领域功能硬编码**：decor_utils、split_post_utils、knowledge_split 中硬编码了教育领域的规则，无通用入口

核心原则：**不增加新概念，只做清理 + 开配置口**。

## CLI 接口变更

### `proseproof convert`

```
proseproof convert <input> -o <output>
    --no-images              剥离所有图片引用
    --strip-small-images     过滤文件大小 <5KB 的图片
    --strip-images-below N   指定字节阈值
```

### `proseproof run` / `proseproof proofread`

`--middleware` 标志现在**真正生效**（覆盖 config.json）：

```
proseproof run input.md --middleware similarity     # 仅启用 Similarity
proseproof run input.md --middleware pre_check      # 仅启用 PreCheck
proseproof run input.md                             # 沿用 config.json 默认
```

## 配置变更

`config.json` 新增/变更字段：

```json
{
  "lecture_split": {
    "skip_sections": []
  },
  "split": {
    "outline": {
      "max_depth": 4,
      "extra_signals": []
    }
  }
}
```

- `skip_sections`：正则模式列表，匹配片段首行的目录在拆分后丢弃。默认空（不跳过）
- `extra_signals`：字符串列表（正则），匹配到的行作为额外的结构信号参与大纲构建和分割边界决策。默认空
