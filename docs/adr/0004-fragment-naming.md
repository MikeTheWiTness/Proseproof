# ADR-0004: Fragment 目录命名

- **状态**: 已采纳
- **日期**: 2026-07-02

## 背景

校对系统将长文档拆分为"片段"（Fragment）进行逐段校对。原有的命名方案源于 K12（中小学教育）场景，使用"第N题"风格的命名（如 `第1题/`、`第2题/`）。随着系统通用化，需要一种语言无关、领域无关的命名方式，适用于学术论文、技术文档、法律文书等各种场景。

## 决策

采用**零填充数字命名**方案：

- **目录命名**: `frag_001/`、`frag_002/`、…、`frag_NNN/`，使用三位零填充数字（不足三位时左侧补零）。
- **内容文件**: 每个片段目录下的主内容文件命名为 `frag_001.md`（与目录名一致）。
- **批量处理**: 校对命令遍历所有匹配 `frag_*/` 模式的目录，按数字顺序处理。

## 后果

**正面**:
- 与国际化和编程惯例一致（零填充数字排序在文件系统中表现为自然顺序）。
- 易于排序和批量处理，`ls frag_*/` 即可按序返回。
- 语言无关、领域无关，适用于任何类型的文档。
- 目录名和文件名一致，降低混淆。

**负面**:
- 碎片数量超过 999 时需考虑是否扩展位数（如升级为四位 `frag_0001/`），但 999 个片段已远超实际需求。
- 从数字编号无法直接推断片段内容，需打开文件或依赖额外的索引。
- 从旧命名迁移需要重命名脚本，但此工作一次性完成即可。
