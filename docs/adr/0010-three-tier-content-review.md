# ADR-0010: 内容审查三层体系

- **状态**: 已采纳
- **日期**: 2026-07-02

## 背景

Proseproof v0.1.0 的校对是「逐片段」的——每个片段独立发送给 LLM，片段之间
互不知晓。这导致一类系统性的漏检：

- **结构级盲区**：章节顺序颠倒、编号跳号、同级标题层级不一致——片段内的
  LLM 看不到全局结构。
- **语义级盲区**：第 3 章说主角已婚、第 1 章说主角单身（事实矛盾）；第 5 章
  使用的术语在第 2 章是另一个名字（术语不一致）——片段 LLM 看不到跨章节的
  语义裂缝。

需要一个「文档级」审查阶段来覆盖这些盲区。但不能简单地将全文扔给 LLM——
长文档超出上下文窗口，且 LLM 注意力随上下文增长而衰减。

## 决策

在 Proofread 和 Typeset 之间插入**内容审查阶段**，分为三层，按成本递增、
信息量递增：

**第一层 — 结构审查（Structural Review）**：
- 输入：`_outline.json`
- 方法：纯 Python 规则检查
- 检查维度：章节/片段顺序、编号连续性（1 → 2 → 3 无跳号）、层级一致性
  （### 不应直接出现在 # 下）、同级标题格式一致性
- LLM 成本：零
- 发现严重问题时**默认暂停，等待用户确认**。可通过 `--yes` 跳过暂停。

**第二层 — Light 审查（A-light）**：
- 输入：`_outline.json` + 每个片段的校对摘要（Proofread 阶段顺手产出）
- 方法：单次 LLM 调用
- 能覆盖：章节顺序、编号连续性、论点遗漏（大纲承诺了 5 个论点但摘要中只
  出现 3 个）、术语不一致（摘要粒度允许时）
- 不能确定：事实矛盾、论证跳跃（摘要信息量不足）
- 产物：`confidence` 字段上限为 `medium`

**第三层 — Full 审查（A-full）**：
- 输入：`_outline.json` + 全文原文 + 所有校对报告
- 方法：单次 LLM 调用（可能受限于上下文窗口）
- 能覆盖：所有 Light 的检查维度 + 事实矛盾、论证跳跃、术语不一致的全文本
  交叉验证
- 产物：`confidence` 字段可有 `high`
- 仅当用户主动选择（`--review full`）时执行

默认层级为 Light（`--review light`），可通过 `--review off` 关闭内容审查。

校对摘要由 Proofread 阶段的 LLM 在校对报告末尾顺手产出（≤50 字），不增加
额外 API 调用。若 LLM 遗漏摘要字段，由格式校验机制检测到后，通过 bash 工具
补写，不触发完整的重新校对。

## 后果

**正面**:
- 覆盖了 v0.1.0 的文档级盲区：结构错误和跨章节语义矛盾。
- 成本可控：默认 Light 模式只增加一次 LLM 调用（基于大纲和摘要，非全文）。
- 结构审查零成本（Python 规则），可无条件执行。
- 三层之间的信息量和 confidence 有明确梯度，用户知道每个结论的可靠程度。

**负面**:
- Light 审查的准确性依赖于校对摘要的质量——如果摘要不准确或遗漏关键信息，
  审查结论可能是误导性的。
- Full 审查在长文档上可能超出上下文窗口，需要额外的分块策略（当前未实现）。
- 审查阶段引入了新的 LLM 调用，增加了总体延迟——对于追求速度的用户，
  需要通过 `--review off` 明确选择跳过。
