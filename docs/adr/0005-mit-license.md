# ADR-0005: 项目许可证

- **状态**: 已采纳
- **日期**: 2026-07-02

## 背景

ProofreadAgent 项目计划以开源形式发布，需要选择一个合适的开源许可证。许可证的选择将影响社区贡献者的参与意愿、商业用户的使用方式以及项目的长期发展。需要选择一种简单、宽松、广为人知的许可证，降低使用和贡献的门槛。

## 决策

采用 **MIT License**（MIT 许可证）。

MIT License 是最广泛使用的开源许可证之一，具有以下特点：

- **简单**: 全文仅约 170 词，易于阅读和理解。
- **宽松**: 允许任何人自由使用、复制、修改、合并、发布、分发、再许可和/或销售软件的副本。
- **免责**: 软件按"原样"提供，不提供任何形式的明示或暗示担保。
- **保留版权声明**: 唯一要求是在所有副本或实质性部分中保留版权声明和许可声明。

## 后果

**正面**:
- 极低的采用门槛，个人和企业均可放心使用。
- 与 GPL 等 copyleft 许可证兼容（MIT 代码可被合并到 GPL 项目中）。
- 社区贡献者无需担心许可证条款的复杂性。
- 有利于项目在学术和商业场景中被广泛引用和集成。

**负面**:
- 无法防止他人将代码用于闭源商业产品而不回馈社区。
- 不提供专利授权（与 Apache 2.0 相比），但由于项目性质（校对工具），专利风险较低。
- 贡献者无法确保其贡献保持开源，但 MIT 的宽松性本身就鼓励最大化的使用和传播。
